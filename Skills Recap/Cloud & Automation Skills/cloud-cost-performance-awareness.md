# Cloud Cost & Performance Awareness (Recap & Application)

## Overview
Cost and performance awareness ensures that cloud solutions remain secure, efficient, and financially sustainable.  
Security and optimisation must coexist to maintain compliance without waste.

## Core Skills Recapped
- **Cost Optimisation:** Using AWS Budgets and Cost Explorer.  
- **Performance Tuning:** Matching instance types to workloads.  
- **Right-Sizing & Auto Scaling:** Efficient resource allocation.  
- **Monitoring & Alerts:** Using CloudWatch for utilisation trends.  
- **Balancing Security vs Cost:** Choosing appropriate security tools without over-provisioning.

## Tools Used
- **AWS Budgets / Cost Explorer** – Expense tracking.  
- **AWS CloudWatch** – Performance monitoring.  
- **Trusted Advisor** – Cost and performance recommendations.  
- **AWS Config** – Compliance vs efficiency review.

## Practical Application / Lab Work
- Analysed EC2 utilisation to identify idle instances.  
- Configured AWS Budgets alerts for resource cost spikes.  
- Compared encryption and logging costs for large data volumes.  
- Tested scaling policies under load scenarios.

## SBS Relevance
For SBS, balancing cost and performance ensures:  
- Reliable customer portal access.  
- Predictable expenses for secure data operations.  
- Sustainable compliance with security and data governance.

## Reflection
Understanding performance and cost helps justify security investments.  
Efficient systems are easier to defend and maintain.
